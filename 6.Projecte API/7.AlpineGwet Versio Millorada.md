# Introducció a Alpine i visualització de dades GET

## Versió perfecta per a 1r DAW

Després de crear la nostra API amb Express i SQLite, ara farem el pas següent:

```text
Base de dades → API → JSON → Alpine → HTML
```

És a dir:

* la base de dades guarda la informació
* Express crea una API
* la API retorna JSON
* Alpine rep eixes dades
* el navegador les mostra en pantalla

Això és una pràctica molt important perquè uneix:

* backend
* frontend
* JavaScript
* HTML dinàmic

# Què és Alpine

**Alpine.js** és una llibreria de JavaScript molt lleugera.

Servix per donar comportament dinàmic al HTML sense necessitat d’un framework gran com Angular o React.

La idea és:

```text
HTML + Alpine = pàgines dinàmiques senzilles
```

Per exemple, amb Alpine podem:

* mostrar text
* ocultar elements
* repetir llistes
* reaccionar a botons
* carregar dades d’una API

---

# Què és una API

Una API és una URL que retorna dades.

Per exemple:

```text
http://localhost:3000/api/ciutats
```

No retorna una pàgina web.

Retorna dades en format JSON.

Exemple:

```json
[
  {
    "ID": 1,
    "NOM": "València"
  }
]
```

# Pas 8. Fer la petició GET

```javascript
window.ciutatsApp = function() {
    return {
        ciutats: [],

        carregarCiutats() {
            fetch('/api/ciutats')
                .then(function(resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutats = dades;
                });
        }
    };
};
```

---

# Explicació lenta de `fetch`

## `fetch('/api/ciutats')`

Demana dades al servidor.

## `.then(...)`

Quan arribe la resposta, continua.

## `resposta.json()`

Converteix el JSON en objectes JavaScript.

## `this.ciutats = dades`

Guarda les dades dins del component Alpine.

---

# Què és `this`

Ací:

```javascript
this
```

vol dir:

```text
el component actual d’Alpine
```

Per això:

```javascript
this.ciutats = dades;
```

guarda l’array dins del component.

---

# Per què usem `=>`

```javascript
(dades) => {
```

Perquè manté el valor correcte de `this`.

# Pas 10. Mostrar en llista

```html
<div x-data="ciutatsApp()" x-init="carregarCiutats()">

    <ul>
        <template x-for="ciutat in ciutats" :key="ciutat.ID">
            <li>
                <strong x-text="ciutat.NOM"></strong>
                -
                <span x-text="ciutat.DESCRIPCIO"></span>
                -
                <span x-text="ciutat.POBLACIO"></span>
            </li>
        </template>
    </ul>

</div>
```

---

# Què és `x-for`

Recorre un array.

```html
x-for="ciutat in ciutats"
```

Vol dir:

```text
per cada element de l’array ciutats
guarda’l en la variable ciutat
```

---

# Què és `<template>`

És una plantilla invisible.

No es veu directament.

Alpine la copia una vegada per cada ciutat.

---

# Què és `:key`

```html
:key="ciutat.ID"
```

És un identificador únic.

Ajuda Alpine a distingir cada fila.

Com el DNI de cada element.

---

# Pas 11. Mostrar en taula

```html
<div x-data="ciutatsApp()" x-init="carregarCiutats()">

<table class="primary">

    <thead>
        <tr>
            <th>ID</th>
            <th>Nom</th>
            <th>Descripció</th>
            <th>Població</th>
        </tr>
    </thead>

    <template x-for="ciutat in ciutats" :key="ciutat.ID">
        <tbody>
            <tr>
                <td x-text="ciutat.ID"></td>
                <td x-text="ciutat.NOM"></td>
                <td x-text="ciutat.DESCRIPCIO"></td>
                <td x-text="ciutat.POBLACIO"></td>
            </tr>
        </tbody>
    </template>

</table>

</div>
```

---

# Per què `tbody`

En taules, el navegador és estricte.

Així Alpine crea blocs correctes.
