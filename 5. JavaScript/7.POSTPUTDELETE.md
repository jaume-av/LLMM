


# Rutes POST, PUT i DELETE

En esta part anem a ampliar el projecte.

Fins ara només hem treballat amb **GET**, és a dir, només hem consultat dades.

Ara anem a permetre:

* crear dades → **POST**
* modificar dades → **PUT**
* eliminar dades → **DELETE**

---

## Què farem exactament

En lloc de posar totes les rutes en un únic fitxer (`routes/index.js`), anem a organitzar millor el projecte.

Cada tipus de dades tindrà el seu fitxer:

* `paisos.js`
* `provincies.js`
* `ciutats.js`

Això ens permet:

* tindre el codi més ordenat
* facilitar el manteniment
* evitar fitxers massa grans

---

# PAS 1: Crear fitxer per a països

Dins de la carpeta:

```text
routes
```

crea un nou fitxer:

```text
paisos.js
```

---

## Contingut inicial de `paisos.js`

Copia este codi:

```javascript
var express = require('express');
var router = express.Router();
const db = require('../db');

router.get('/', function(req, res) {
  db('PAIS')
    .select('*')
    .then(function(rows) {
      res.json(rows);
    });
});

module.exports = router;
```

---

## Què fa este fitxer

Ara la ruta:

```text
/api/paisos
```

ja no estarà en `index.js`, sinó en este nou fitxer.

---

# PAS 2: Connectar el nou fitxer en `app.js`

Obri el fitxer:

```text
app.js
```

Busca la zona on es carreguen les rutes.

Afig:

```javascript
var paisosRouter = require('./routes/paisos');
```

I després afig:

```javascript
app.use('/api/paisos', paisosRouter);
```

---

## Important entendre açò

Abans teníem:

```javascript
router.get('/api/paisos', ...)
```

Ara tenim:

```javascript
router.get('/', ...)
```

Perquè la part `/api/paisos` ja la posa `app.js`.

---

# PAS 3: Provar que tot funciona igual

Arranca el servidor:

```bash
npm start
```

I prova:

```text
http://localhost:3000/api/paisos
```

Si tot està bé, hauria de funcionar igual que abans.

# PAS 4: Afegir POST

Ara anem a crear dades noves.

## Afegix en `paisos.js`

```javascript
router.post('/', function(req, res) {
  const nouPais = req.body;

  db('PAIS')
    .insert(nouPais)
    .then(function() {
      res.json({ missatge: 'País creat' });
    });
});
```

---

# PAS 5: Permetre rebre JSON

Perquè POST funcione, necessitem dir-li a Express que accepte JSON.

Obri `app.js` i afig:

```javascript
app.use(express.json());
```

# PAS 7: Afegir PUT

Per modificar un registre.

## Afig en `paisos.js`

```javascript
router.put('/:id', function(req, res) {
  const id = req.params.id;
  const dades = req.body;

  db('PAIS')
    .where('ID', id)
    .update(dades)
    .then(function() {
      res.json({ missatge: 'País actualitzat' });
    });
});
```

---

# PAS 8: Provar PUT

```text
PUT http://localhost:3000/api/paisos/1
```

Body:

```json
{
  "NOM": "Espanya modificada"
}
```

---

# PAS 9: Afegir DELETE

Per eliminar un registre.

## Afig en `paisos.js`

```javascript
router.delete('/:id', function(req, res) {
  const id = req.params.id;

  db('PAIS')
    .where('ID', id)
    .del()
    .then(function() {
      res.json({ missatge: 'País eliminat' });
    });
});
```

---

# PAS 10: Provar DELETE

```text
DELETE http://localhost:3000/api/paisos/1
```

## Mètodes utilitzats amb Knex

En este projecte hem utilitzat diversos mètodes de **Knex** per treballar amb la base de dades.

A continuació tens un resum clar de cadascun:

---

* **`.select('*')`**
  Obté totes les files d’una taula.

---

* **`.where(camp, valor)`**
  Filtra els resultats segons una condició.

---

* **`.first()`**
  Retorna només el primer resultat (un únic registre).

---

* **`.insert(dades)`**
  Inserix un nou registre en la taula.

---

* **`.update(dades)`**
  Modifica un registre existent.

---

* **`.del()`**
  Elimina un registre de la taula.

---

* **`.then(...)`**
  S’executa quan la consulta ha funcionat correctament.

---

* **`.catch(...)`**
  S’executa si hi ha un error en la consulta.

---

## Idea important

Hem de tindre en compte que:

* els mètodes com `.select()`, `.insert()`, `.update()` i `.del()` **fan la consulta**
* `.then(...)` i `.catch(...)` **gestionen el resultat**

---

Això ens permet separar clarament:

```text
Consulta → Resultat correcte → Error
```

I és la base de com funcionen totes les rutes en Express amb base de dades.
