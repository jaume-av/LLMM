# PUT

L’operació **PUT** s’utilitza per modificar dades que ja existeixen en una API REST.

A diferència de **POST**, que crea nous registres, **PUT** actualitza un registre existent. Per això és necessari indicar quin element volem modificar, normalment mitjançant el seu identificador `ID`.

En esta part afegirem la primera operació **PUT** del projecte. Utilitzarem esta operació per modificar les dades d’una ciutat que ja existeix a la base de dades.

---

Fins ara només havíem treballat amb:

```text
GET /api/ciutats
```

per llegir dades de l’API, i amb:

```text
POST /api/ciutats
```

per crear ciutats noves.

Ara utilitzarem **PUT** per modificar dades que ja existeixen a l’API.

La nova ruta serà:

```text
PUT /api/ciutats/:id
```

Esta ruta necessita un paràmetre `:id` per saber quina ciutat volem modificar. Per exemple:

```text
PUT /api/ciutats/3
```

significa:

```text
modifica la ciutat que té ID 3
```

Amb **POST** creàvem una ciutat nova, mentre que amb **PUT** modificarem una ciutat que ja existeix. Per això necessitem indicar el seu identificador únic `ID`.

---

En resum, els passos que realitzarem són:

1.- En `routes/ciutats.js` crearem una nova **ruta PUT** per poder modificar una ciutat existent.

2.- Dins d’esta mateixa ruta afegirem l’`update` de **Knex** per actualitzar les dades en **SQLite**.

3.- En el fitxer de les vistes crearem `vistaModificarCiutat()` per mostrar el formulari d’edició.

4.- També carregarem les dades de la ciutat seleccionada amb `fetch` per omplir automàticament el formulari.

5.- Després crearem `modificarCiutatApp()` per controlar el formulari amb AlpineJS.

6.- Quan es guarden els canvis, enviarem una petició PUT amb `fetch` al servidor.

7.- Modificarem el botó **“Modificar”** de la taula perquè òbriga la vista d’edició.

8.- Finalment, tornarem a la llista de ciutats per veure les dades actualitzades.

## 1. Crear la ruta PUT en `routes/ciutats.js`

El primer pas serà preparar el backend perquè puga rebre una petició de modificació.

En el fitxer:

```text
routes/ciutats.js
```

afegirem una nova ruta abans de:

```javascript
module.exports = router;
```

La ruta serà esta:

```javascript
router.put('/:id', function(req, res) {

});
```

Esta ruta respondrà a peticions com esta:

```text
PUT /api/ciutats/3
```

El valor `3` arribarà al servidor mitjançant:

```javascript
req.params.id
```

Això ens permet saber quina ciutat volem modificar.

De moment només hem creat l’estructura de la ruta. En el pas següent afegirem dins d’ella l’`update` de Knex per actualitzar les dades en SQLite.

## 2. Afegir l’`update` de Knex dins de la ruta PUT

Ara que ja tenim creada la ruta PUT, afegirem dins d’ella el codi que modifica les dades en la base de dades.

En el fitxer:

```text
routes/ciutats.js
```

la ruta quedarà així:

```javascript
router.put('/:id', function(req, res) {
    db('CIUTAT')
        .where('ID', req.params.id)
        .update({
            NOM: req.body.NOM,
            DESCRIPCIO: req.body.DESCRIPCIO,
            IMATGE: req.body.IMATGE,
            POBLACIO: req.body.POBLACIO,
            PROVINCIA_ID: req.body.PROVINCIA_ID
        })
        .then(function(resultat) {
            res.json({ missatge: 'Ciutat modificada correctament' });
        })
        .catch(function(error) {
            res.status(500).json({ error: 'Error en la base de dades' });
        });
});
```

Amb esta part indiquem tres coses:

```javascript
db('CIUTAT')
```

treballem amb la taula `CIUTAT`.

```javascript
.where('ID', req.params.id)
```

busquem la ciutat que té l’ID que arriba en la ruta.

```javascript
.update({...})
```

actualitzem els camps amb les dades que arriben des del formulari.

Per exemple, si fem esta petició:

```text
PUT /api/ciutats/3
```

el servidor modificarà la ciutat que té:

```text
ID = 3
```

i li posarà les noves dades rebudes en `req.body`.


## 3. Crear `vistaModificarCiutat()`

Anirem construint esta vista a poc a poc. Primer crearem l’estructura bàsica i després l’anirem completant.

Ara prepararem la vista que mostrarà el formulari de modificació.

En el fitxer:

```text
public/js/ciutats-vista.js
```

afegirem una funció nova:

```javascript
function vistaModificarCiutat() {
    return `
    
    `;
}
```

Esta vista serà molt pareguda a `vistaNovaCiutat()`, però en lloc de crear una ciutat nova servirà per editar una ciutat existent.

La idea és mostrar un formulari amb els camps:

```text
NOM
DESCRIPCIO
POBLACIO
PROVINCIA_ID
IMATGE
```

i omplir-los automàticament amb les dades actuals de la ciutat seleccionada.

De moment només crearem l’estructura de la vista. En el següent pas carregarem les dades amb `fetch`.


## 4. Carregar les dades de la ciutat amb `fetch`

Ara necessitem carregar les dades de la ciutat seleccionada per omplir automàticament el formulari.

Per exemple, si volem modificar la ciutat amb ID `3`, farem una petició a:

```text
/api/ciutats/3
```

Això farà que el servidor ens retorne les dades d’eixa ciutat.

Per preparar-ho, modificarem la vista perquè reba l’`id` de la ciutat:

```javascript
function vistaModificarCiutat(id) {
```

i també passarem l’ID a Alpine:

```html
<section x-data="modificarCiutatApp(${id})">
```

La funció quedarà així:

```javascript
function vistaModificarCiutat(id) {
    return `
        <section x-data="modificarCiutatApp(${id})">

            <h2>Modificar ciutat</h2>

            <form @submit.prevent="modificarCiutat()">

                <label for="nom">Nom</label>
                <input id="nom" type="text" x-model="ciutat.NOM">

                <label for="descripcio">Descripció</label>
                <input id="descripcio" type="text" x-model="ciutat.DESCRIPCIO">

                <label for="poblacio">Població</label>
                <input id="poblacio" type="number" x-model="ciutat.POBLACIO">

                <label for="provincia">Província</label>
                <input id="provincia" type="number" x-model="ciutat.PROVINCIA_ID">

                <label for="imatge">Imatge</label>
                <input id="imatge" type="text" x-model="ciutat.IMATGE">

                <p>
                    <button type="submit">Guardar canvis</button>

                    <button type="button"
                            class="warning"
                            onclick="mostrarCiutats()">
                        Cancel·lar
                    </button>
                </p>

            </form>

        </section>
    `;
}
```

En el següent pas crearem `modificarCiutatApp(id)` i farem el `fetch` per carregar les dades de la ciutat.

En AlpineJS, cada x-data sol estar controlat per una app diferent.


## 5. Crear `modificarCiutatApp()`

Ara crearem l’app Alpine que controlarà el formulari de modificació.

En el fitxer:

```text
public/js/ciutats-vista.js
```

afegirem este codi:

```javascript
window.modificarCiutatApp = function (id) {
    return {
        ciutat: {
            NOM: '',
            DESCRIPCIO: '',
            IMATGE: '',
            POBLACIO: '',
            PROVINCIA_ID: ''
        },

        carregarCiutat() {
            fetch('/api/ciutats/' + id)
                .then(function (resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutat = dades;
                });
        }
    };
};
```

Esta funció rep l’`id` de la ciutat que volem modificar.

Després, amb:

```javascript
fetch('/api/ciutats/' + id)
```

demanem al servidor les dades d’eixa ciutat.

I quan arriben les dades, les guardem en:

```javascript
this.ciutat = dades;
```

Com el formulari utilitza `x-model`, els camps s’ompliran automàticament.


## 6. Executar `carregarCiutat()` automàticament

Ara mateix ja tenim creada la funció:

```javascript
carregarCiutat()
```

però encara no s’executa.

Necessitem que Alpine la llance automàticament quan es carregue la vista de modificació.

Per fer-ho, modificarem esta línia:

```html
<section x-data="modificarCiutatApp(${id})">
```

i afegirem:

```html
x-init="carregarCiutat()"
```

La línia quedarà així:

```html
<section x-data="modificarCiutatApp(${id})"
         x-init="carregarCiutat()">
```

Ara, quan obrim la vista de modificació:

```javascript
vistaModificarCiutat(3)
```

Alpine executarà automàticament:

```javascript
carregarCiutat()
```

i farà el `fetch` a:

```text
/api/ciutats/3
```

Després, els camps del formulari s’ompliran automàticament amb les dades de la ciutat seleccionada.

El codi complet de `vistaModificarCiutat(id)` quedarà així:

```javascript
function vistaModificarCiutat(id) {
    return `
        <section x-data="modificarCiutatApp(${id})"
                 x-init="carregarCiutat()">

            <h2>Modificar ciutat</h2>

            <form @submit.prevent="modificarCiutat()">

                <label for="nom">Nom</label>
                <input id="nom" type="text" x-model="ciutat.NOM">

                <label for="descripcio">Descripció</label>
                <input id="descripcio" type="text" x-model="ciutat.DESCRIPCIO">

                <label for="poblacio">Població</label>
                <input id="poblacio" type="number" x-model="ciutat.POBLACIO">

                <label for="provincia">Província</label>
                <input id="provincia" type="number" x-model="ciutat.PROVINCIA_ID">

                <label for="imatge">Imatge</label>
                <input id="imatge" type="text" x-model="ciutat.IMATGE">

                <p>
                    <button type="submit">Guardar canvis</button>

                    <button type="button"
                            class="warning"
                            onclick="mostrarCiutats()">
                        Cancel·lar
                    </button>
                </p>

            </form>

        </section>
    `;
}
```


## 7. Crear `modificarCiutat()` dins de `modificarCiutatApp()`

Ara afegirem la funció que enviarà la petició PUT al servidor per guardar els canvis.

En el fitxer:

```text
public/js/ciutats-vista.js
```

dins de:

```javascript
window.modificarCiutatApp = function (id) {
```

afegirem:

```javascript
modificarCiutat() {
    fetch('/api/ciutats/' + id, {
        method: 'PUT',
        headers: {
            'Content-Type': 'application/json'
        },
        body: JSON.stringify(this.ciutat)
    })
        .then(function (resposta) {
            return resposta.json();
        })
        .then(function () {
            mostrarCiutats();
        });
}
```

El resultat final quedarà així:

```javascript
window.modificarCiutatApp = function (id) {
    return {
        ciutat: {
            NOM: '',
            DESCRIPCIO: '',
            IMATGE: '',
            POBLACIO: '',
            PROVINCIA_ID: ''
        },

        carregarCiutat() {
            fetch('/api/ciutats/' + id)
                .then(function (resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutat = dades;
                });
        },

        modificarCiutat() {
            fetch('/api/ciutats/' + id, {
                method: 'PUT',
                headers: {
                    'Content-Type': 'application/json'
                },
                body: JSON.stringify(this.ciutat)
            })
                .then(function (resposta) {
                    return resposta.json();

                .then(function () {

```

Quan polsem:

```text
Guardar canvis
```

s’enviarà una petició:

```text
PUT /api/ciutats/3
```

amb les dades modificades de la ciutat.

## 8. Modificar el botó `Modificar` de la taula

En `vistaCiutats()` canviarem:

```html
<button class="success">Modificar</button>
```

per:

```html
<button type="button"
        class="success"
        @click="mostrarModificarCiutat(ciutat.ID)">
    Modificar
</button>
```

És important utilitzar:

```html
@click=""
```

i no:

```html
onclick=""
```

perquè `ciutat.ID` és una variable d’AlpineJS.

## 9. Crear `mostrarModificarCiutat(id)` en `spa.js`

En el fitxer:

```text
public/js/spa.js
```

afegirem:

```javascript
function mostrarModificarCiutat(id) {
    document.getElementById("spa").innerHTML = vistaModificarCiutat(id);
    Alpine.initTree(document.getElementById("spa"));
}
```

Esta funció carregarà la vista de modificació dins del contenidor principal:

```html
<div id="spa"></div>
```

i reinicialitzarà AlpineJS perquè el formulari funcione correctament.

## 10. Resultat final

Ara el flux complet funcionarà així:

```text
Llista de ciutats
→ Modificar
→ Formulari d’edició
→ Guardar canvis
→ PUT /api/ciutats/:id
→ Tornar a la llista
```

Quan es guarden els canvis, la taula es tornarà a carregar amb les dades actualitzades.





# DELETE

En esta part afegirem l’operació **DELETE** del projecte.

A diferència de:

```text
POST
```

que crea dades noves, o:

```text
PUT
```

que modifica dades existents, **DELETE** elimina un registre de la base de dades.

La ruta que utilitzarem serà:

```text
DELETE /api/ciutats/:id
```

Per exemple:

```text
DELETE /api/ciutats/3
```

significa:

```text
elimina la ciutat que té ID 3
```

---

## Resum del que farem

1.- Crearem la ruta DELETE en `routes/ciutats.js`.

2.- Afegirem el `.delete()` de Knex per eliminar la ciutat.

3.- Modificarem el botó **Eliminar** de la taula.

4.- Crearem la funció `eliminarCiutat(id)` en AlpineJS.

5.- Enviarem la petició DELETE amb `fetch`.

6.- Tornarem a carregar la llista de ciutats.

---



## Crear la ruta DELETE  en `routes/ciutats.js`

Al fitxer:

```text
routes/ciutats.js
```

Afegir abans de:

```javascript
module.exports = router;
```

```javascript
router.delete('/:id', function(req, res) {
    db('CIUTAT')
        .where('ID', req.params.id)
        .delete()
        .then(function(resultat) {
            res.json({ missatge: 'Ciutat eliminada correctament' });
        })
        .catch(function(error) {
            res.status(500).json({ error: 'Error en la base de dades' });
        });
});
```

---


## Modificar el botó `Eliminar` de la taula en `public/js/ciutats-vista.js`

Canviar:

```html
<button class="error">Eliminar</button>
```

per:

```html
<button type="button"
        class="error"
        @click="eliminarCiutat(ciutat.ID)">
    Eliminar
</button>
```

---

## Afegir `eliminarCiutat(id)` dins de `ciutatsApp()`


En este cas no crearem una app nova com en **POST** o **PUT**, perquè l’eliminació es fa directament des de la mateixa vista de la taula. No necessitem obrir un formulari nou ni canviar de vista. El botó **Eliminar** ja està dins de `vistaCiutats()`, així que afegirem la funció `eliminarCiutat(id)` dins de la mateixa `ciutatsApp()` que controla la taula.

El botó **Eliminar** ja està dins de:

```html
<section x-data="ciutatsApp()">
```

Per tant, la funció:

```javascript
eliminarCiutat(id)
```

també ha d’estar dins de:

```javascript
window.ciutatsApp = function () {
```

A diferència de **POST** i **PUT**, DELETE no necessita un formulari ni una vista nova. Només elimina la ciutat seleccionada i torna a carregar la taula.

* **Per tant:**

Afegir dins de:

```javascript
window.ciutatsApp = function () {
```

```javascript
eliminarCiutat(id) {
    fetch('/api/ciutats/' + id, {
        method: 'DELETE'
    })
        .then(function (resposta) {
            return resposta.json();
        })
        .then(() => {
            this.carregarCiutats();
        });
}
```

---

## Resultat final dins de `ciutatsApp()`

```javascript
window.ciutatsApp = function () {
    return {
        ciutats: [],

        carregarCiutats() {
            fetch('/api/ciutats')
                .then(function (resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutats = dades;
                });
        },

        eliminarCiutat(id) {
            fetch('/api/ciutats/' + id, {
                method: 'DELETE'
            })
                .then(function (resposta) {
                    return resposta.json();
                })
                .then(() => {
                    this.carregarCiutats();
                });
        }
    };
};
```

Ara, quan polsem:

```text
Eliminar
```

s’enviarà una petició:

```text
DELETE /api/ciutats/:id
```

i la ciutat desapareixerà de la taula després de recarregar les dades.
