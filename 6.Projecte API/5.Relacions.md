# Relacions entre recursos

En esta part veurem com relacionar els recursos entre si.

Per exemple:

* un país té moltes províncies
* una província té moltes ciutats
* un usuari pot tindre una ciutat associada

Això ens permetrà fer consultes més complexes i mostrar dades relacionades en la nostra aplicació.

Fins ara, quan consultàvem les ciutats, obteníem una resposta simple amb els camps de la taula `CIUTAT`.

Per exemple, una ciutat podia tindre:

* el seu `ID`
* el seu `NOM`
* la seua `DESCRIPCIO`
* la seua `POBLACIO`
* el seu `PROVINCIA_ID`

El problema és que `PROVINCIA_ID` només és un número.

Per a una persona que consulta la web, vore:

```text
PROVINCIA_ID = 3
```

no és tan útil com vore:

```text
PROVINCIA_NOM = Castelló
```

Per això, en esta part farem un pas més:

```text
CIUTAT + PROVINCIA → dades més útils
```

L’objectiu serà que la nostra API no retorne només l’identificador de la província, sinó també el seu nom.

I després mostrarem eixa informació en una taula HTML amb Alpine.

---

## Què volem aconseguir

Partim d’una situació com esta.

### Abans

La ruta:

```text
/api/ciutats
```

retornava dades de la taula `CIUTAT` sense relacionar.

Exemple:

```json
[
  {
    "ID": 1,
    "NOM": "Castelló de la Plana",
    "DESCRIPCIO": "Capital de la província de Castelló",
    "IMATGE": "castello.jpg",
    "POBLACIO": 170000,
    "PROVINCIA_ID": 1
  }
]
```

Açò funciona, però la columna `PROVINCIA_ID` no és gaire clara per a l’usuari.

### Després

Volem que la mateixa ruta retorne també el nom de la província:

```json
[
  {
    "ID": 1,
    "NOM": "Castelló de la Plana",
    "DESCRIPCIO": "Capital de la província de Castelló",
    "IMATGE": "castello.jpg",
    "POBLACIO": 170000,
    "PROVINCIA_ID": 1,
    "PROVINCIA_NOM": "Castelló",
    "PAIS_NOM": "Espanya"
  }
]
```

Ara sí tenim una resposta molt més útil.

---

## Idea clau d’esta part

La idea central és esta:

**les relacions és millor resoldre-les en el backend, no en Alpine.**

És a dir:

* Express i Knex consulten la base de dades
* Knex unix les taules relacionades
* la API retorna un JSON més ric
* Alpine només mostra les dades

Per tant, el flux serà:

```text
SQLite → Knex amb JOIN → API → Alpine → HTML
```

---

## Què és una relació entre taules

En bases de dades relacionals, una taula pot tindre un camp que apunte a una altra taula.

En el nostre cas:

* `CIUTAT.PROVINCIA_ID` apunta a `PROVINCIA.ID`
* `PROVINCIA.PAIS_ID` apunta a `PAIS.ID`

Això vol dir:

* cada ciutat pertany a una província
* cada província pertany a un país

Per tant, si volem mostrar informació més completa, necessitem unir les taules.

---

## Què és un JOIN

Un `JOIN` és una operació que permet unir dades de dues o més taules relacionades.

Per exemple:

* taula `CIUTAT`
* taula `PROVINCIA`

Si una ciutat té un `PROVINCIA_ID`, podem unir-la amb la seua província corresponent.

En SQL la idea seria:

```sql
SELECT CIUTAT.NOM, PROVINCIA.NOM
FROM CIUTAT
JOIN PROVINCIA ON CIUTAT.PROVINCIA_ID = PROVINCIA.ID;
```

Amb Knex escriurem una cosa equivalent, però en JavaScript.

# Situació inicial

## Ruta antiga de `routes/ciutats.js`

Abans teníem una versió simple com esta:

```javascript
var express = require('express');
var router = express.Router();
const db = require("../db");


router.get('/', function(req, res) {
    db('CIUTAT').select('*').then(rows => res.json(rows));
});

router.get('/:id', function(req, res) {
    db('CIUTAT').where('ID', req.params.id).first().then(row => res.json(row));
});

module.exports = router;
```

## Què fa esta versió

Esta versió:

* consulta només la taula `CIUTAT`
* retorna tots els camps de la ciutat
* no unix cap altra taula
* per tant, no sap quin nom té la província

Per això només podem mostrar:

```text
PROVINCIA_ID
```

i no:

```text
PROVINCIA_NOM
```

---

# Pas 1. Modificar la ruta GET de totes les ciutats

Ara anem a substituir la consulta simple per una consulta amb relacions.

Deixa `routes/ciutats.js` així:

```javascript
var express = require('express');
var router = express.Router();
const db = require('../db');

router.get('/', function(req, res) {
    db('CIUTAT')
        .join('PROVINCIA', 'CIUTAT.PROVINCIA_ID', '=', 'PROVINCIA.ID')
        .join('PAIS', 'PROVINCIA.PAIS_ID', '=', 'PAIS.ID')
        .select(
            'CIUTAT.ID',
            'CIUTAT.NOM',
            'CIUTAT.DESCRIPCIO',
            'CIUTAT.IMATGE',
            'CIUTAT.POBLACIO',
            'CIUTAT.PROVINCIA_ID',
            'PROVINCIA.NOM as PROVINCIA_NOM',
            'PAIS.NOM as PAIS_NOM'
        )
        .then(function(rows) {
            res.json(rows);
        })
        .catch(function(error) {
            res.status(500).json({ error: 'Error en la base de dades' });
        });
});

router.get('/:id', function(req, res) {
    db('CIUTAT')
        .join('PROVINCIA', 'CIUTAT.PROVINCIA_ID', '=', 'PROVINCIA.ID')
        .join('PAIS', 'PROVINCIA.PAIS_ID', '=', 'PAIS.ID')
        .select(
            'CIUTAT.ID',
            'CIUTAT.NOM',
            'CIUTAT.DESCRIPCIO',
            'CIUTAT.IMATGE',
            'CIUTAT.POBLACIO',
            'CIUTAT.PROVINCIA_ID',
            'PROVINCIA.NOM as PROVINCIA_NOM',
            'PAIS.NOM as PAIS_NOM'
        )
        .where('CIUTAT.ID', req.params.id)
        .first()
        .then(function(row) {
            if (!row) {
                res.status(404).json({ error: 'No trobat' });
            } else {
                res.json(row);
            }
        })
        .catch(function(error) {
            res.status(500).json({ error: 'Error en la base de dades' });
        });
});

module.exports = router;
```

---

# Pas 2. Entendre la consulta nova

Ara anem a veure què fa cada part.

## Unir `CIUTAT` amb `PROVINCIA`

```javascript
.join('PROVINCIA', 'CIUTAT.PROVINCIA_ID', '=', 'PROVINCIA.ID')
```

Açò vol dir:

* busca la província de cada ciutat
* compara el camp `CIUTAT.PROVINCIA_ID`
* amb el camp `PROVINCIA.ID`

Quan coincidixen, Knex sap quina província correspon a cada ciutat.

---

## Unir `PROVINCIA` amb `PAIS`

```javascript
.join('PAIS', 'PROVINCIA.PAIS_ID', '=', 'PAIS.ID')
```

Açò fa un segon pas:

* a partir de la província trobada
* busca el país al qual pertany

Així, cada ciutat podrà portar també informació del país.

---

## Triar exactament quins camps volem

```javascript
.select(
    'CIUTAT.ID',
    'CIUTAT.NOM',
    'CIUTAT.DESCRIPCIO',
    'CIUTAT.IMATGE',
    'CIUTAT.POBLACIO',
    'CIUTAT.PROVINCIA_ID',
    'PROVINCIA.NOM as PROVINCIA_NOM',
    'PAIS.NOM as PAIS_NOM'
)
```

Açò indica que volem retornar:

* els camps principals de la ciutat
* el nom de la província
* el nom del país

### Què significa `as`

Per exemple:

```javascript
'PROVINCIA.NOM as PROVINCIA_NOM'
```

vol dir:

* agafa el camp `NOM` de la taula `PROVINCIA`
* però en el JSON final posa-li el nom `PROVINCIA_NOM`

Això és molt útil perquè tant `CIUTAT` com `PROVINCIA` tenen un camp `NOM`, i necessitem distingir-los.

# Pas 4. Revisar `public/js/ciutats.js`

En esta part no necessitem grans canvis.

El fitxer pot quedar així:

```javascript
window.ciutatsApp = function() {
    return {
        ciutats: [],

        carregarCiutats() {
            fetch('/api/ciutats')
                .then(function(resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutats = dades;
                });
        }
    };
};
```

## Per què no cal canviar-lo molt

Perquè `fetch('/api/ciutats')` continua sent la mateixa ruta.

La diferència és que ara la API retorna un JSON més complet.

Per tant:

* Alpine continua fent la mateixa petició
* però rep més informació
* i la podem mostrar en pantalla

# Pas 5. Crear la taula HTML

Ara en `public/ciutats.html` deixarem una taula que mostre els camps més útils.

Deixa el fitxer així:

```html
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Llista de ciutats</title>

    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/picnic/7.1.0/picnic.min.css">
    <script src="js/ciutats.js" defer></script>
    <script defer src="https://cdnjs.cloudflare.com/ajax/libs/alpinejs/3.15.0/cdn.min.js"></script>
</head>
<body>

<main class="container">

    <h1>Llista de ciutats</h1>

    <div x-data="ciutatsApp()" x-init="carregarCiutats()">
        <table class="table">
            <thead>
            <tr>
                <th>ID</th>
                <th>Nom</th>
                <th>Descripció</th>
                <th>Població</th>
                <th>Província</th>
                <th>Imatge</th>
            </tr>
            </thead>
            <tbody>
            <template x-for="ciutat in ciutats" :key="ciutat.ID">
                <tr>
                    <td x-text="ciutat.ID"></td>
                    <td x-text="ciutat.NOM"></td>
                    <td x-text="ciutat.DESCRIPCIO"></td>
                    <td x-text="ciutat.POBLACIO"></td>
                    <td x-text="ciutat.PROVINCIA_NOM"></td>
                    <td x-text="ciutat.IMATGE"></td>
                </tr>
            </template>
            </tbody>
        </table>
    </div>

</main>

</body>
</html>
```

---

# Pas 6. Entendre el canvi en el HTML

La part clau és esta:

```html
<td x-text="ciutat.PROVINCIA_NOM"></td>
```

Abans mostràvem una cosa així:

```html
<td x-text="ciutat.PROVINCIA_ID"></td>
```

Això ensenyava només un número.

Ara mostrem:

```html
ciutat.PROVINCIA_NOM
```

i això és molt més útil perquè l’usuari veu directament el nom de la província.
