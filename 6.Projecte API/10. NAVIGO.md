# Introducció a Navigo

Fins ara hem construït la nostra SPA canviant manualment el contingut de:

```html
<div id="spa"></div>
```

Per exemple:

```javascript
function mostrarCiutats() {
    document.getElementById("spa").innerHTML = vistaCiutats();
    Alpine.initTree(document.getElementById("spa"));
}
```

Este sistema funciona correctament i ens ha permés entendre com funciona una SPA des de zero:

* carregar vistes dinàmicament
* treballar amb `fetch`
* utilitzar AlpineJS
* implementar CRUD
* separar frontend i backend

Però conforme el projecte creix, apareixen alguns problemes.

---

## Problemes de la SPA actual

Actualment tota l’aplicació funciona sempre en la mateixa URL:

```text
/
```

Encara que estiguem veient:

* ciutats
* països
* províncies
* formularis de modificació

la URL no canvia.

Això provoca diverses limitacions.

### No podem compartir una vista concreta

Per exemple, no existeix una URL com:

```text
#/ciutats
```

o:

```text
#/ciutats/modificar/3
```

Per tant, no podem copiar l’enllaç exacte d’una vista.

---

### El navegador no recorda la navegació

Els botons:

```text
enrere
endavant
```

del navegador no funcionen correctament com en una SPA real.

---

>Si refresquem la pàgina, podem perdre la vista actual

Per exemple, si estem modificant una ciutat i fem:

```text
F5
```

la vista es reinicia.

---

## Què és Navigo

Navigo és una llibreria JavaScript molt lleugera que permet gestionar rutes dins d’una SPA.

Un router és el component encarregat de decidir:

```text
quina vista mostrar segons la URL
```

Per exemple:

```text
#/ciutats
#/paisos
#/provincies
```

Cada URL carregarà automàticament una vista diferent.

---

### Per a què serveix un router

Un router ens permet convertir la nostra SPA manual en una SPA més organitzada i professional.

En lloc de cridar manualment:

```javascript
mostrarCiutats();
mostrarPaisos();
mostrarProvincies();
```

podrem definir rutes:

```javascript
router.on('/ciutats', ...)
router.on('/paisos', ...)
router.on('/provincies', ...)
```

Així, la navegació passarà a dependre de la URL.

---

### Avantatges d’utilitzar Navigo

1. **URLs reals per a cada vista**

Per exemple:

```text
#/ciutats
#/paisos
#/provincies
```

o inclús:

```text
#/ciutats/modificar/3
```

---

2. **Navegació més professional**

El navegador podrà gestionar correctament:

* historial
* enrere
* endavant

---

3. **Millor organització del projecte**

La navegació deixarà d’estar dispersa en botons manuals.

Ara existirà una estructura clara de rutes.

---

4. **Facilita projectes més grans**

Quan el projecte tinga moltes vistes, formularis o apartats, un router ajuda a mantindre el codi ordenat.

---

5. **Aproxima el projecte als frameworks moderns**

Frameworks com:

* Vue
* React
* Angular

utilitzen routers interns.

Navigo ens permet entendre esta idea sense necessitat d’utilitzar encara frameworks més complexos.

---

### Per què té sentit incorporar-lo ara

Ara sí que el projecte ja té sentit com a SPA real perquè ja hem implementat:

* GET
* POST
* PUT
* DELETE
* canvi de vistes
* AlpineJS
* Fetch API
* CRUD complet

Ja entenem com funciona la navegació manual.

Per això, incorporar ara un router és una evolució natural del projecte, no una “màgia” automàtica.

---

### Idea principal

Fins ara nosaltres decidíem manualment:

```javascript
document.getElementById("spa").innerHTML = ...
```

Amb Navigo continuarem carregant vistes dinàmicament, però ara serà el router qui decidirà:

```text
quina vista mostrar segons la URL actual
```

Això fa que la nostra SPA siga:

* més escalable
* més organitzada
* més semblant a una aplicació moderna real
* més fàcil de mantindre conforme cresca el projecte.


### Resum dels passos que realitzarem

1.- Instal·larem la llibreria Navigo en el projecte.

2.- Afegirem el fitxer JavaScript de Navigo a `index.html`.

3.- Crearem el router principal en `spa.js`.

4.- Definirem les rutes principals de l’aplicació.

5.- Substituirem funcions com `mostrarCiutats()` per navegació amb rutes.

6.- Farem que cada ruta carregue la seua vista corresponent.

7.- Afegirem rutes amb paràmetres, com ara modificar una ciutat concreta.

8.- Adaptarem els botons perquè naveguen utilitzant el router.

9.- Farem que la URL canvie automàticament segons la vista actual.

10.- Finalment, convertirem la nostra SPA manual en una SPA amb routing real.


---


# Integració de Navigo en el projecte

A partir d’ara partim de l’estructura actual del projecte:

```text
public/
├── index.html
└── js/
    ├── spa.js
    ├── ciutats-vista.js
    ├── paisos-vista.js
    └── provincies-vista.js
```

I també partim del contenidor principal:

```html
<div id="spa"></div>
```

La idea és mantindre les vistes que ja tenim, però fer que la navegació depenga de **Navigo** i no de cridades manuals com `mostrarCiutats()`. Açò encaixa amb l’evolució del projecte des de la SPA manual que ja havíeu plantejat. 

---

### 1. Afegir Navigo en `index.html`

En el fitxer:

```text
public/index.html
```

afegirem la llibreria Navigo abans dels nostres scripts.

La zona dels scripts hauria de quedar així:

```html
<script src="https://unpkg.com/navigo"></script>

<script src="js/ciutats-vista.js" defer></script>
<script src="js/paisos-vista.js" defer></script>
<script src="js/provincies-vista.js" defer></script>
<script src="js/spa.js" defer></script>

<script defer
        src="https://cdnjs.cloudflare.com/ajax/libs/alpinejs/3.15.0/cdn.min.js">
</script>
```

És important que `spa.js` vaja després dels fitxers de vistes, perquè `spa.js` utilitzarà funcions com:

```javascript
vistaCiutats()
vistaPaisos()
vistaProvincies()
```

---

### 2. Crear una funció comuna per carregar vistes

En el fitxer:

```text
public/js/spa.js
```

crearem una funció per no repetir sempre el mateix codi.

```javascript
function carregarVista(vista) {
    document.getElementById("spa").innerHTML = vista;
    Alpine.initTree(document.getElementById("spa"));
}
```

Fins ara fèiem açò en cada funció:

```javascript
document.getElementById("spa").innerHTML = vistaCiutats();
Alpine.initTree(document.getElementById("spa"));
```

Ara ho centralitzem en una sola funció.

---

### 3. Crear el router principal

En `spa.js`, després de `carregarVista()`, crearem el router:

```javascript
const router = new Navigo('/');
```

Este objecte serà l’encarregat de controlar les rutes de la nostra SPA.

---

### 4. Crear les rutes principals

Ara definirem les rutes principals de l’aplicació:

```javascript
router.on('/ciutats', function () {
    carregarVista(vistaCiutats());
});

router.on('/paisos', function () {
    carregarVista(vistaPaisos());
});

router.on('/provincies', function () {
    carregarVista(vistaProvincies());
});
```

Ara, quan la ruta siga:

```text
#/ciutats
```

es carregarà la vista de ciutats.

Quan siga:

```text
#/paisos
```

es carregarà la vista de països.

I quan siga:

```text
#/provincies
```

es carregarà la vista de províncies.

---

### 5. Afegir una ruta inicial

També indicarem què ha de passar quan l’usuari entre a la pàgina sense cap ruta concreta.

```javascript
router.on('/', function () {
    router.navigate('/ciutats');
});
```

Així, si obrim:

```text
http://localhost:3000
```

l’aplicació ens portarà directament a la vista de ciutats.

---

### 6. Activar el router

Al final de `spa.js`, afegirem:

```javascript
router.resolve();
```

Esta línia fa que Navigo llija la URL actual i carregue la vista corresponent.

---

### 7. Primer resultat de `spa.js`

De moment, `spa.js` pot quedar així:

```javascript
function carregarVista(vista) {
    document.getElementById("spa").innerHTML = vista;
    Alpine.initTree(document.getElementById("spa"));
}

const router = new Navigo('/');

router.on('/', function () {
    router.navigate('/ciutats');
});

router.on('/ciutats', function () {
    carregarVista(vistaCiutats());
});

router.on('/paisos', function () {
    carregarVista(vistaPaisos());
});

router.on('/provincies', function () {
    carregarVista(vistaProvincies());
});

router.resolve();
```

En este punt ja tenim les primeres rutes funcionant.

---

### 8. Modificar els botons principals

En `index.html`, els botons ja no haurien de cridar directament:

```javascript
mostrarCiutats()
mostrarPaisos()
mostrarProvincies()
```

Ara han de navegar amb el router.

Canvia:

```html
<button onclick="mostrarCiutats()">Ciutats</button>
<button class="success" onclick="mostrarPaisos()">Països</button>
<button class="warning" onclick="mostrarProvincies()">Províncies</button>
```

per:

```html
<button onclick="router.navigate('/ciutats')">Ciutats</button>
<button class="success" onclick="router.navigate('/paisos')">Països</button>
<button class="warning" onclick="router.navigate('/provincies')">Províncies</button>
```

Ara els botons canvien la URL i Navigo carrega la vista corresponent.
