# Introducció a Alpine i visualització de dades GET

Després de refactoritzar les rutes GET, el projecte ja té una estructura més clara:

* `index.js` per a les rutes generals
* `paisos.js` per als països
* `provincies.js` per a les províncies
* `ciutats.js` per a les ciutats

Ara farem el pas següent:

```text
API → JSON → Alpine → HTML
```

És a dir:

* el navegador farà una petició a la API
* rebrà dades en format JSON
* i les mostrarem dins d’una pàgina web

En esta part treballarem amb **Alpine**, una llibreria de JavaScript molt lleugera que ens permet fer pàgines dinàmiques sense muntar un framework gran.

L’objectiu serà visualitzar en una pàgina web les dades que ja retornen les nostres rutes GET.

En esta primera pràctica amb Alpine treballarem sobre la ruta:

```text
/api/ciutats
```

i mostrarem en pantalla la llista de ciutats.

---

## Què és Alpine

**Alpine** és una llibreria de JavaScript que ens permet afegir comportament dinàmic a una pàgina HTML d’una manera molt simple.

La seua idea és pareguda a esta:

* el HTML continua sent el centre de la pàgina
* Alpine ens deixa afegir dades i comportament directament sobre eixe HTML
* no necessitem una configuració gran
* no necessitem compilar res

Per això és una molt bona opció per a un projecte com el nostre.

Amb Alpine podem:

* guardar dades en variables
* mostrar dades en el HTML
* recórrer arrays
* cridar funcions
* reaccionar quan la pàgina es carrega
* connectar fàcilment amb la nostra API

---

## Què farem exactament amb Alpine

En esta pràctica volem fer una cosa concreta:

* crear una pàgina HTML
* carregar Alpine
* crear un component
* cridar la ruta GET `/api/ciutats`
* guardar les dades rebudes
* mostrar-les en una llista

Per tant, treballarem amb estos conceptes d’Alpine:

* `x-data`
* `x-init`
* `x-for`
* `x-text`

I també treballarem amb:

* `fetch()`
* `.then(...)`
* `resposta.json()`

---

## Què significa cada peça que utilitzarem

### `x-data`

Servix per crear un component Alpine.

És la part on definim:

* les dades
* les funcions
* el comportament del component

Exemple:

```html
<div x-data="ciutatsApp()">
```

Açò vol dir que este tros de HTML estarà controlat per la funció `ciutatsApp()`.

---

### `x-init`

Servix per executar una funció quan el component es crea.

Exemple:

```html
<div x-data="ciutatsApp()" x-init="carregarCiutats()">
```

Açò significa que, en carregar-se el component, s’executarà la funció `carregarCiutats()`.

---

### `x-for`

Servix per recórrer una llista de dades.

Exemple:

```html
<template x-for="ciutat in ciutats">
```

Açò significa que Alpine recorrerà l’array `ciutats` i repetirà el contingut del `template` una vegada per cada element.

---

### `x-text`

Servix per mostrar un valor dins d’un element HTML.

Exemple:

```html
<span x-text="ciutat.NOM"></span>
```

Açò farà que dins del `span` aparega el valor de `ciutat.NOM`.

---

## Com funciona el procés complet

El procés que seguirem serà este:

```text
1. El navegador obri ciutats.html
2. Alpine es carrega
3. Alpine crea el component
4. Alpine executa carregarCiutats()
5. fetch() demana /api/ciutats
6. Express consulta SQLite
7. la API retorna JSON
8. Alpine guarda les dades en un array
9. el HTML es redibuixa automàticament
```

Esta és la idea clau de tota esta part.

## On posarem els fitxers

Com que Express ja servix fitxers estàtics des de `public`, treballarem dins d’eixa carpeta.

Crearem esta estructura:

```text
public/
├── ciutats.html
└── js
    └── ciutats.js
```

### Què farà cada fitxer

#### `public/ciutats.html`

Contindrà:

* l’estructura HTML
* la càrrega d’Alpine
* la zona on es mostraran les ciutats

#### `public/js/ciutats.js`

Contindrà:

* la funció Alpine
* la crida a la API
* la lògica per guardar les dades

Això ens permet separar HTML i JavaScript i deixar el codi més clar.

---

## Pas 4. Instal·lació o càrrega d’Alpine

En este projecte no farem una instal·lació amb npm per al frontend. Per a esta pràctica utilitzarem Alpine carregant-lo directament des d’un CDN.

Dins de `<head>`, afegix estes línies:

```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/picnic/7.1.0/picnic.min.css">
<script defer src="https://cdnjs.cloudflare.com/ajax/libs/alpinejs/3.15.0/cdn.min.js"></script>
```

Ara `ciutats.html` quedarà així:

```html
<!DOCTYPE html>
<html lang="ca">
<head>
    <meta charset="UTF-8">
    <title>Llista de ciutats</title>

    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/picnic/7.1.0/picnic.min.css">
    <script defer src="https://cdnjs.cloudflare.com/ajax/libs/alpinejs/3.15.0/cdn.min.js"></script>
</head>
<body>

<h1>Llista de ciutats</h1>

</body>
</html>
```

* **Este codi:**

Carrega Picnic CSS i Alpine des d’internet.

La paraula `defer` indica que el navegador esperarà a tindre el HTML carregat abans d’executar el script.

Açò és convenient perquè evita problemes d’ordre de càrrega.

## Pas 10. Crear la funció que crida la API

Ara amplia `public/js/ciutats.js` així:

```javascript
window.ciutatsApp = function() {
    return {
        ciutats: [],

        carregarCiutats() {
            fetch('/api/ciutats')
                .then(function(resposta) {
                    return resposta.json();
                })
                .then(function(dades) {
                    console.log(dades);
                });
        }
    };
};
```

---

## Què fa esta funció

### `fetch('/api/ciutats')`

Fa una petició GET a la ruta:

```text
/api/ciutats
```

### `resposta.json()`

Converteix la resposta HTTP en dades JavaScript.

### `console.log(dades)`

Mostra les dades en la consola del navegador.

De moment no les estem pintant encara. Només comprovem que arriben bé.

## Pas 13. Guardar les dades dins del component

Ara canvia la funció de `ciutats.js` així:

```javascript
window.ciutatsApp = function() {
    return {
        ciutats: [],

        carregarCiutats() {
            fetch('/api/ciutats')
                .then(function(resposta) {
                    return resposta.json();
                })
                .then((dades) => {
                    this.ciutats = dades;
                });
        }
    };
};
```

## En resum

En esta part hem aprés a:

* carregar Alpine en una pàgina
* crear un component
* crear una funció pròpia
* fer una petició GET amb `fetch`
* rebre JSON de la nostra API
* guardar eixes dades en un array
* mostrar-les en el HTML amb Alpine

---

Hem de quedar-nos amb esta idea:

**Alpine no substituïx la API. Alpine consumeix la API i mostra les dades en el navegador.**

El flux complet ara és:

```text
Navegador → Alpine → fetch → API Express → SQLite → JSON → Alpine → HTML
```
