# Refactorització de les rutes GET

> **Refactoritzar** és el procés de **modificar el codi** per millorar la seua estructura interna **sense canviar** el seu comportament.

En esta part volem millorar l’organització del projecte abans de començar a treballar amb Alpine.

* traure les rutes GET de països, províncies i ciutats de `index.js`
* repartir-les en fitxers separats
* mantindre el mateix funcionament que ja tenim

L’objectiu és que el projecte continue funcionant igual, però amb una estructura més clara.

### Per què convé fer esta refactorització

Quan totes les rutes estan en un únic fitxer:

* costa més localitzar cada bloc
* el fitxer es fa massa llarg
* es barreja la lògica de recursos diferents
* després és més difícil ampliar el projecte

Si separem les rutes per recursos, cada fitxer tindrà una responsabilitat molt clara.

Això ens aporta estos avantatges:

* el codi queda més ordenat
* és més fàcil llegir-lo
* és més fàcil mantindre’l
* ens prepara millor per a la fase següent amb Alpine

### Com organitzarem les rutes GET

Dins de la carpeta:

```text
routes
```

crearem estos fitxers nous:

```text
routes/
├── index.js
├── paisos.js
├── provincies.js
├── ciutats.js
└── users.js
```

### `routes/index.js`

Es quedarà només amb les rutes generals:

* `/`
* `/api`
* `/api/test`

### `routes/paisos.js`

Contindrà:

* GET de tots els països
* GET de país per ID

### `routes/provincies.js`

Contindrà:

* GET de totes les províncies
* GET de província per ID

### `routes/ciutats.js`

Contindrà:

* GET de totes les ciutats
* GET de ciutat per ID

# Fitxers que has de crear

Dins de `routes`, crea estos tres fitxers:

```text
paisos.js
provincies.js
ciutats.js
```

## Canvis al fitxer `app.js`

La refactorització consistirà en tres passos molt clars:

### 1. Deixar `index.js` només amb les rutes generals

El fitxer `routes/index.js` quedarà així:

```javascript
var express = require('express');
var router = express.Router();

router.get('/', function(req, res) {
  res.send('Servidor funcionant');
});

router.get('/api', function(req, res) {
  res.send('OK');
});

router.get('/api/test', function(req, res) {
  res.json({
    missatge: 'API funcionant correctament',
    any: 2026,
    centre: 'IES Benigasló'
  });
});

module.exports = router;
```

### 2. Moure les rutes GET a fitxers separats

**1. `routes/paisos.js`**

```javascript
var express = require('express');
var router = express.Router();
const db = require('../db');

router.get('/', function(req, res) {
  db('PAIS')
    .select('*')
    .then(function(rows) {
      res.json(rows);
    });
});

router.get('/:id', function(req, res) {
  db('PAIS')
    .where('ID', req.params.id)
    .first()
    .then(function(row) {
      res.json(row);
    });
});

module.exports = router;
```

**2. `routes/provincies.js`**

```javascript
var express = require('express');
```

### 3. Modificar `app.js`

Ara hem de dir-li a Express que utilitze estos nous fitxers.

En `app.js`, busca la zona on es carreguen les rutes.
Segurament tindràs una cosa semblant a:

```javascript
var indexRouter = require('./routes/index');
var usersRouter = require('./routes/users');
```

Canvia-ho per:

```javascript
var indexRouter = require('./routes/index');
var usersRouter = require('./routes/users');
var paisosRouter = require('./routes/paisos');
var provinciesRouter = require('./routes/provincies');
var ciutatsRouter = require('./routes/ciutats');
```

> Nota: `usersRouter` es manté perquè forma part de l’estructura del projecte i es pot utilitzar més avant per gestionar usuaris.

Després busca la zona on es connecten les rutes.
Segurament tindràs una cosa semblant a:

```javascript
app.use('/', indexRouter);
app.use('/users', usersRouter);
```

Canvia-ho per:

```javascript
app.use('/', indexRouter);
app.use('/users', usersRouter);
app.use('/api/paisos', paisosRouter);
app.use('/api/provincies', provinciesRouter);
app.use('/api/ciutats', ciutatsRouter);
```

### Nota important

Ara, dins dels fitxers nous, les rutes són:

```javascript
router.get('/')
router.get('/:id')
```

i no:

```javascript
router.get('/api/paisos')
router.get('/api/paisos/:id')
```

Això és així perquè la part:

```text
/api/paisos
```

ja la posa `app.js` amb:

```javascript
app.use('/api/paisos', paisosRouter);
```

El mateix passa amb `provincies` i `ciutats`.

### Comprovació

Quan guardes els fitxers, reinicia el servidor i comprova que continuen funcionant exactament igual estes rutes:

* `http://localhost:3000/api/paisos`
* `http://localhost:3000/api/paisos/1`
* `http://localhost:3000/api/provincies`
* `http://localhost:3000/api/provincies/1`
* `http://localhost:3000/api/ciutats`
* `http://localhost:3000/api/ciutats/1`

Si tot està bé, el comportament serà el mateix que abans, però amb el codi molt més ordenat.

## En resum

No estem canviant la funcionalitat del projecte.
Només estem millorant l’organització del codi.

Això ens permetrà continuar després amb Alpine d’una manera més clara, perquè ja tindrem:

* `index.js` per a rutes generals
* `paisos.js` per a països
* `provincies.js` per a províncies
* `ciutats.js` per a ciutats
